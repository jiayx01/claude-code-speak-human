# 数据产品经理场景

这个 output style 最适合下面这些场景。

## 1. 需求评审

```text
业务要加一个“急售”标签，说能提升点击率。
请帮我准备需求评审时该问的问题和判断标准。
```

理想回答：

- 先判断需求最大风险
- 问清用户价值、口径、供给真实性、滥用风险
- 不只看 CTR，要看咨询率、带看转化、投诉等护栏指标

## 2. 异常分析

```text
昨天 APP 首页点击率突然下降 18%，曝光量基本没变，转化率也下降。
请给我一个 30 分钟内能推进的异常分析路径。
```

理想回答：

- 先给初判
- 先排数据/埋点，再拆流量结构和产品变化
- 明确 30 分钟内能定位到哪一层，不承诺一步找到根因

## 3. PRD 改写

```text
请把这段需求改成更适合 PRD 的表达：
用户点进房源详情后，如果这个房源质量比较高，就给经纪人一个提醒，让他尽快联系用户，不然用户可能流失。
```

理想回答：

- 直接给可粘贴进 PRD 的表达
- 补齐目标、触发条件、展示对象、展示内容、规则、验收标准
- 把“质量比较高”“尽快”改成可配置、可验收的口径

## 4. 技术方案转述

```text
研发说要把实时推荐链路从同步调用改成异步消息队列，可能会降低接口耗时，但会带来一致性问题。
请帮我转成业务方听得懂的版本。
```

理想回答：

- 少用技术术语
- 讲清收益、代价、影响范围和决策点
- 不牺牲技术准确性
