# 实验设计

这个项目来自一个小型 prompt / output style 实验。

## 背景

观察到的问题：

- Opus 4.8 的代码能力、上下文能力更强
- 但中文表达有时更难读
- Opus 4.6 的中文表达和逻辑呈现更自然

目标不是训练模型，而是通过配置让 Opus 4.8 尽量获得更好的中文表达。

## 对照组

实验设计了四组：

| 组别 | 说明 |
| --- | --- |
| A | Opus 4.6 原始输出，作为参考风格 |
| B | Opus 4.8 原始输出，作为 baseline |
| C | Opus 4.8 + `CLAUDE.md` |
| D | Opus 4.8 + `CLAUDE.md` + `Human Chinese` output style |

## 评分维度

每项 1-5 分，共 50 分：

1. 中文自然度
2. 结论清晰度
3. 逻辑连贯性
4. 层次结构
5. 用户可读性
6. 技术准确性
7. 信息密度
8. 不确定性表达
9. 可执行性
10. 简洁度

## Pilot 结果

| 任务 | Opus 4.6 | 4.8 baseline | 4.8 + CLAUDE.md | 4.8 full | 胜出 |
| --- | ---: | ---: | ---: | ---: | --- |
| T05 异常分析 | 37 | 42 | 44 | 43 | 4.8 + CLAUDE.md |
| T06 需求评审 | 36 | 45 | 44 | 46 | 4.8 full |
| T07 PRD 改写 | 36 | 44 | 43 | 45 | 4.8 full |
| 平均 | 36.3 | 43.7 | 43.7 | 44.7 | 4.8 full |

说明：Opus 4.6 的相关段落中文质量不差，但在本次 CLI pilot 中出现了跨任务过度回答，因此整体可用性被扣分。最终配置吸收它的自然表达，但增加了“不要过度展开”的防护。

## 结论

最优组合是：

```text
Opus 4.8 + 短 CLAUDE.md + Human Chinese output style
```

它没有明显牺牲能力，但能改善中文表达的自然度、结论清晰度和可读性。
