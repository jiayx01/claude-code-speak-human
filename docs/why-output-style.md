# 为什么用 Output Style

很多人遇到中文表达问题后，第一反应是往 `CLAUDE.md` 里塞更多规则。

这通常会带来两个问题：

1. `CLAUDE.md` 越长，模型越容易变成“按模板答题”。
2. 表达风格、项目上下文、个人偏好混在一起，后面很难调。

更好的分工是：

| 文件 | 负责什么 | 不负责什么 |
| --- | --- | --- |
| `CLAUDE.md` | 身份、长期偏好、项目约定 | 不负责细致语气和格式控制 |
| `output style` | 语气、表达、结构、中文可读性 | 不负责项目上下文 |

这个项目的核心思路是：

> Claude 已经很聪明，不需要被管死；我们只是帮它把中文表达校准到更自然、更清楚。

所以 `CLAUDE.md` 要短，`output style` 也要克制。

## 不要过度约束

不推荐这样写：

```text
你必须永远按以下 12 个步骤回答，每次都输出目标、背景、方案、风险、指标、行动项……
```

这种写法短期看起来整齐，长期会让回答越来越像模板。

更推荐这样写：

```text
第一段先给核心判断。
复杂问题要有清楚的递进关系。
不要为了显得完整而过度展开。
```

它不会替 Claude 做所有判断，只是在中文表达上给一个方向。
